# 草稿目录

这个目录存放所有正在进行中的草稿文章。

## 使用方式

1. **创建新草稿：**
   ```bash
   # 从模板复制
   cp templates/article-template.md drafts/2025-11-20-my-new-article.md
   ```

2. **编辑草稿：**
   - 在这个目录中自由编辑
   - 可以随时保存未完成的工作
   - 使用 Git 追踪版本变化

3. **完成草稿：**
   ```bash
   # 将完成的文章移动到正式目录
   mv drafts/2025-11-20-my-article.md docs/articles/
   ```

## 文件组织

- 使用标准文件命名：`YYYY-MM-DD-主题.md`
- 保持草稿数量合理，定期清理
- 长期未完成的草稿考虑归档或删除

## 草稿状态

草稿可以有以下状态（在文档元数据中标记）：
- **草稿：** 刚开始写作
- **进行中：** 正在积极编写
- **待审阅：** 写作完成，等待审阅
- **暂停：** 暂时搁置

## 提示

💡 定期回顾这个目录，确保草稿不会积压太久。

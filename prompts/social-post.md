# 社群貼文生成提示

## 使用時機
將研究內容轉換為社群媒體貼文時使用。

---

## 提示詞模板

```
請將以下研究內容轉換為社群貼文：

[貼上研究內容或檔案路徑]

**貼文要求：**
- 目標字數：1,500-2,500 字
- 適合 Facebook/LinkedIn 閱讀
- 吸引人的開頭（hook）
- 清晰的段落結構
- 結尾有明確的觀點或啟示

**風格指南：**
- 參考 WRITING_GUIDE.md 的商業寫作風格章節
- 敘事性強，避免過多條列式
- 適當使用金句和比喻
- 每段不超過 4 行
- 可以有個人觀點和立場

**結構建議：**
1. 開頭 hook（引起注意）
2. 背景說明（為什麼重要）
3. 核心內容（2-3 個主要論點）
4. 結論/觀點（給讀者的啟示）
5. 標籤（#相關標籤）

請生成貼文，並在最後附上建議的 hashtags。
```

---

## 變體

### 極短版（Twitter/X 適用）
```
將以下內容濃縮為 280 字以內的推文：

[內容]

要求：
- 一句話核心觀點
- 數據或金句支撐
- 引發討論的結尾
```

### 系列貼文（Thread）
```
將以下內容拆分為 5-7 則推文串（Thread）：

[內容]

要求：
- 第 1 則：吸引人的 hook
- 中間則：每則一個重點
- 最後一則：總結 + call to action
- 每則可獨立閱讀，但串在一起更完整
```

### LinkedIn 專業版
```
請以專業但親和的語調，將以下內容改寫為 LinkedIn 貼文：

[內容]

要求：
- 開頭用個人觀察或問題引入
- 分享 3 個專業洞見
- 結尾邀請讀者分享看法
- 適當使用 emoji（但不要過多）
```

---

## 品質檢查清單

生成貼文後，確認：
- [ ] 開頭 3 句話內要抓住注意力
- [ ] 有明確的「所以呢？」（So what?）
- [ ] 適合目標讀者（商業/科技愛好者）
- [ ] 沒有過多專業術語
- [ ] 結尾有行動呼籲或引發討論的問題

## 檔案命名規則

### 基本原則

- **全部使用小寫字母**
- **使用連字號 `-` 分隔單詞**（不使用底線或空格）
- **包含日期前綴**（格式：`YYYY-MM-DD`）
- **簡潔且描述性強**

### 命名格式

```
YYYY-MM-DD-主題關鍵詞.md
```

### 範例

✅ **正確範例：**
- `2025-11-20-claude-writing-guide.md`
- `2025-11-20-javascript-async-await.md`
- `2025-11-20-productivity-tips.md`

❌ **錯誤範例：**
- `Claude Writing Guide.md`（包含空格和大寫）
- `2025_11_20_guide.md`（使用底線）
- `writing-guide.md`（缺少日期）
- `2025-11-20-Guide.md`（包含大寫字母）

### 特殊情況

- **系列文章：** `2025-11-20-series-name-part-01.md`
- **筆記：** `2025-11-20-note-topic.md`
- **教學：** `2025-11-20-tutorial-topic.md`

## Heading 規則

### 層級結構

```markdown
# H1 - 文章標題（每篇文章只能有一個）

## H2 - 主要章節

### H3 - 子章節

#### H4 - 細分主題

##### H5 - 更細的分類（謹慎使用）

###### H6 - 最細分類（儘量避免）
```

### 使用原則

1. **H1 標題：**
   - 每篇文章只有一個 H1
   - H1 是文章的主標題
   - 應該清晰描述文章主題

2. **H2 標題：**
   - 用於劃分文章的主要章節
   - 應該邏輯清晰，層次分明

3. **H3-H6 標題：**
   - 用於更細的內容劃分
   - 避免層級過深（建議不超過 H4）

### 標題格式規範

```markdown
✅ 正確格式：
# 文章標題

## 第一章節

### 子主題

❌ 錯誤格式：
#文章標題（# 後缺少空格）
##第一章節（## 後缺少空格）
```

### 標題命名建議

- 使用清晰、描述性的標題
- 避免過長的標題（建議不超過 50 字元）
- 使用動詞開頭更具行動性（如「理解」、「掌握」、「實現」）
- 保持同級標題的語法一致性

## 文章結構

### 標準文章結構

```markdown
# 文章標題

> 簡短的文章摘要或引言（可選）

## 元資料

- **建立日期：** YYYY-MM-DD
- **更新日期：** YYYY-MM-DD
- **標籤：** #tag1 #tag2 #tag3
- **狀態：** 草稿/已發布

## 目錄

（對於長文章，可選）

## 引言

文章的開頭，說明寫作目的和背景。

## 主要內容

### 章節 1

內容...

### 章節 2

內容...

## 總結

總結文章要點。

## 參考資料

- 參考連結 1
- 參考連結 2

## 相關文章

- [相關文章 1](連結)
- [相關文章 2](連結)
```

### 文章類型模板

#### 1. **技術文章**
- 問題描述
- 解決方案
- 程式碼範例
- 最佳實踐
- 總結

#### 2. **教學**
- 前置知識
- 步驟詳解
- 實作練習
- 常見問題
- 下一步學習

#### 3. **筆記**
- 主題
- 要點記錄
- 個人思考
- 待深入研究

## 內容規範

### 語言風格

- **清晰簡潔：** 避免冗長複雜的句子
- **主動語態：** 優先使用主動語態
- **一致性：** 保持術語和表達方式的一致
- **讀者導向：** 考慮讀者的背景和需求

### 段落規範

- 每個段落專注一個主題
- 段落之間空一行
- 避免過長的段落（建議 3-5 句）

### 列表使用

```markdown
✅ 有序列表（步驟、優先順序）：
1. 第一步
2. 第二步
3. 第三步

✅ 無序列表（並列要點）：
- 要點 1
- 要點 2
- 要點 3

✅ 任務列表：
- [ ] 待完成任務
- [x] 已完成任務
```

## Markdown 格式規範

### 程式碼區塊

````markdown
```語言名稱
程式碼內容
```

範例：

```javascript
const greeting = "Hello, World!";
console.log(greeting);
```
````

### 行內程式碼

使用反引號包裹：`` `程式碼` ``

範例：使用 `console.log()` 輸出資訊。

### 連結

```markdown
[連結文字](URL)

範例：[Claude AI](https://claude.ai)
```

### 圖片

```markdown
![圖片描述](圖片URL)

範例：![範例圖片](./images/example.png)
```

### 引用

```markdown
> 這是一段引用文字
> 可以多行
```

### 強調

```markdown
**粗體文字**
*斜體文字*
~~刪除線~~
```

### 表格

```markdown
| 欄1 | 欄2 | 欄3 |
|-----|-----|-----|
| 資料1 | 資料2 | 資料3 |
| 資料4 | 資料5 | 資料6 |
```

### 水平線

```markdown
---
```

## 元資料規範

### 文章元資料

每篇文章應在開頭包含以下元資料：

```markdown
## 元資料

- **建立日期：** 2025-11-20
- **更新日期：** 2025-11-20
- **作者：** [你的名字]
- **標籤：** #技術 #寫作 #Claude
- **狀態：** 草稿
- **字數：** 約 2000 字
```

### 標籤系統

使用標籤分類文章：

- **技術類：** `#程式設計` `#網頁開發` `#資料科學`
- **工具類：** `#Claude` `#Git` `#VSCode`
- **方法類：** `#教學` `#指南` `#最佳實踐`
- **主題類：** `#生產力` `#學習` `#思考`

### 狀態標記

- **草稿：** 正在寫作中
- **審閱中：** 等待審閱
- **已發布：** 完成並發布
- **歸檔：** 已過時的內容

## 品質檢查清單

在完成文章後，使用以下清單檢查：

- [ ] 檔案命名符合規範
- [ ] 只有一個 H1 標題
- [ ] 標題層級合理
- [ ] 包含元資料資訊
- [ ] 段落清晰，邏輯連貫
- [ ] 程式碼區塊有語言標註
- [ ] 連結都可用
- [ ] 拼寫和語法正確
- [ ] 格式一致

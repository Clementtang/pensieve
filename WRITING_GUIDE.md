## 文件命名规则

### 基本原则

- **全部使用小写字母**
- **使用连字符 `-` 分隔单词**（不使用下划线或空格）
- **包含日期前缀**（格式：`YYYY-MM-DD`）
- **简洁且描述性强**

### 命名格式

```
YYYY-MM-DD-主题关键词.md
```

### 示例

✅ **正确示例：**
- `2025-11-20-claude-writing-guide.md`
- `2025-11-20-javascript-async-await.md`
- `2025-11-20-productivity-tips.md`

❌ **错误示例：**
- `Claude Writing Guide.md`（包含空格和大写）
- `2025_11_20_guide.md`（使用下划线）
- `writing-guide.md`（缺少日期）
- `2025-11-20-Guide.md`（包含大写字母）

### 特殊情况

- **系列文章：** `2025-11-20-series-name-part-01.md`
- **笔记：** `2025-11-20-note-topic.md`
- **教程：** `2025-11-20-tutorial-topic.md`

## Heading 规则

### 层级结构

```markdown
# H1 - 文章标题（每篇文章只能有一个）

## H2 - 主要章节

### H3 - 子章节

#### H4 - 细分主题

##### H5 - 更细的分类（谨慎使用）

###### H6 - 最细分类（尽量避免）
```

### 使用原则

1. **H1 标题：**
   - 每篇文章只有一个 H1
   - H1 是文章的主标题
   - 应该清晰描述文章主题

2. **H2 标题：**
   - 用于划分文章的主要章节
   - 应该逻辑清晰，层次分明

3. **H3-H6 标题：**
   - 用于更细的内容划分
   - 避免层级过深（建议不超过 H4）

### 标题格式规范

```markdown
✅ 正确格式：
# 文章标题

## 第一章节

### 子主题

❌ 错误格式：
#文章标题（# 后缺少空格）
##第一章节（## 后缺少空格）
```

### 标题命名建议

- 使用清晰、描述性的标题
- 避免过长的标题（建议不超过 50 字符）
- 使用动词开头更具行动性（如"理解"、"掌握"、"实现"）
- 保持同级标题的语法一致性

## 文章结构

### 标准文章结构

```markdown
# 文章标题

> 简短的文章摘要或引言（可选）

## 元数据

- **创建日期：** YYYY-MM-DD
- **更新日期：** YYYY-MM-DD
- **标签：** #tag1 #tag2 #tag3
- **状态：** 草稿/已发布

## 目录

（对于长文章，可选）

## 引言

文章的开头，说明写作目的和背景。

## 主要内容

### 章节 1

内容...

### 章节 2

内容...

## 总结

总结文章要点。

## 参考资料

- 参考链接 1
- 参考链接 2

## 相关文章

- [相关文章 1](链接)
- [相关文章 2](链接)
```

### 文章类型模板

#### 1. **技术文章**
- 问题描述
- 解决方案
- 代码示例
- 最佳实践
- 总结

#### 2. **教程**
- 前置知识
- 步骤详解
- 实践练习
- 常见问题
- 下一步学习

#### 3. **笔记**
- 主题
- 要点记录
- 个人思考
- 待深入研究

## 内容规范

### 语言风格

- **清晰简洁：** 避免冗长复杂的句子
- **主动语态：** 优先使用主动语态
- **一致性：** 保持术语和表达方式的一致
- **读者导向：** 考虑读者的背景和需求

### 段落规范

- 每个段落专注一个主题
- 段落之间空一行
- 避免过长的段落（建议 3-5 句）

### 列表使用

```markdown
✅ 有序列表（步骤、优先级）：
1. 第一步
2. 第二步
3. 第三步

✅ 无序列表（并列要点）：
- 要点 1
- 要点 2
- 要点 3

✅ 任务列表：
- [ ] 待完成任务
- [x] 已完成任务
```

## Markdown 格式规范

### 代码块

````markdown
```语言名称
代码内容
```

示例：

```javascript
const greeting = "Hello, World!";
console.log(greeting);
```
````

### 行内代码

使用反引号包裹：`` `代码` ``

示例：使用 `console.log()` 输出信息。

### 链接

```markdown
[链接文字](URL)

示例：[Claude AI](https://claude.ai)
```

### 图片

```markdown
![图片描述](图片URL)

示例：![示例图片](./images/example.png)
```

### 引用

```markdown
> 这是一段引用文字
> 可以多行
```

### 强调

```markdown
**粗体文字**
*斜体文字*
~~删除线~~
```

### 表格

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 数据1 | 数据2 | 数据3 |
| 数据4 | 数据5 | 数据6 |
```

### 水平线

```markdown
---
```

## 元数据规范

### 文章元数据

每篇文章应在开头包含以下元数据：

```markdown
## 元数据

- **创建日期：** 2025-11-20
- **更新日期：** 2025-11-20
- **作者：** [你的名字]
- **标签：** #技术 #写作 #Claude
- **状态：** 草稿
- **字数：** 约 2000 字
```

### 标签系统

使用标签分类文章：

- **技术类：** `#编程` `#web开发` `#数据科学`
- **工具类：** `#Claude` `#Git` `#VSCode`
- **方法类：** `#教程` `#指南` `#最佳实践`
- **主题类：** `#生产力` `#学习` `#思考`

### 状态标记

- **草稿：** 正在写作中
- **审阅中：** 等待审阅
- **已发布：** 完成并发布
- **归档：** 已过时的内容

- [ ] 文件命名符合规范

- [ ] 代码块有语言标注
- [ ] 链接都可用
- [ ] 拼写和语法正确
- [ ] 格式一致

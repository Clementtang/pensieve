# Google UCP vs Stripe/OpenAI ACP 深度比較分析

> 研究日期：2026-01-13
> 背景：Google 於 2026 年 1 月 11 日在 NRF 大會上發布 Universal Commerce Protocol (UCP)

---

## 執行摘要

AI 代理商務（Agentic Commerce）正在重塑電子商務格局。兩大陣營正在競爭定義這個新領域的標準：

1. **Google UCP** (Universal Commerce Protocol) - 與 Shopify 共同開發，專注於完整購物生命週期
2. **OpenAI/Stripe ACP** (Agentic Commerce Protocol) - 專注於交易結帳流程

這兩個協議**並非互斥**，而是代表不同的設計哲學和生態系統策略。

---

| **授權協議** | 開源（具體授權待確認） |

- 支付商：Visa、Mastercard、American Express、Adyen、**Stripe**

## 二、架構設計比較

### 2.1 設計範圍

```
┌─────────────────────────────────────────────────────────────────┐
│                    購物生命週期階段                               │
├─────────┬─────────┬─────────┬─────────┬─────────┬─────────────────┤
│  發現   │  比較   │  決策   │  結帳   │  支付   │  售後服務        │
├─────────┴─────────┴─────────┼─────────┴─────────┼─────────────────┤
│                             │                   │                 │
│         Google UCP          │      ACP          │     UCP         │
│    (完整購物體驗覆蓋)        │  (聚焦交易時刻)    │   (售後支持)     │
│                             │                   │                 │
└─────────────────────────────┴───────────────────┴─────────────────┘
```

| 維度 | UCP | ACP |
|------|-----|-----|

| **設計重心** | 多代理協作、跨平台標準 | 單一對話界面內的交易完成 |
| **抽象層級** | 更高層級（協調多個協議） | 聚焦層級（支付流程優化） |

## 三、支付流程比較

### 3.1 UCP 支付流程

```
用戶 ──► AI 代理 ──► UCP ──► 支付處理商 ──► 令牌化 ──► 返回代理 ──► 結帳閘道 ──► 商家系統
                              │
                              ├── Visa
                              ├── Mastercard
                              ├── Stripe
                              ├── Adyen
                              └── (支付處理商無關)
```

**特點：**
- 支付處理商無關（Payment Processor Agnostic）
- 商家技術棧無關
- 整合 AP2 作為底層支付層
- 使用 Google Pay 完成交易

### 3.2 ACP 支付流程

```
用戶 ──► ChatGPT ──► ACP ──► Stripe ──► 委託支付令牌 ──► 商家
                              │
                              └── 單次使用
                                  時間限制
                                  金額限制
```

**特點：**
- Stripe 為主要支付處理商
- 委託支付令牌（Delegated Payment Tokens）機制
- 已有 Stripe 商家可一行代碼啟用
- PayPal 支持即將推出

## 四、關鍵技術差異

### 4.1 代理協作能力

| 能力 | UCP | ACP |
|------|-----|-----|
| **多代理協作** | ✅ 原生支持 | ❌ 單代理設計 |
| **代理間通訊** | 透過 A2A 協議 | 不適用 |
| **代理-商家認證** | ✅ 內建 | ❌ 無此組件 |
| **跨平台代理** | ✅ 設計目標 | ❌ ChatGPT 中心 |

**UCP 代理協作範例：**
```
代理 A (搜索比較) ──► UCP ──► 代理 B (結帳支付)
         │                          │
         └──────── A2A 協議 ────────┘
```

### 4.2 整合複雜度

| 整合方式 | UCP | ACP |
|----------|-----|-----|
| **REST API** | ✅ | ✅ |
| **MCP** | ✅ | ✅ |
| **A2A** | ✅ | ❌ |
| **最小整合** | 待確認 | 1 行代碼（已有 Stripe） |

### 4.3 數據與隱私

| 項目 | UCP | ACP |
|------|-----|-----|
| **商家記錄身份** | 商家 | 商家 |
| **客戶關係** | 商家擁有 | 商家擁有 |
| **支付憑證** | 令牌化處理 | 委託令牌 |

## 六、零售商整合建議

### 6.1 支持兩個協議

**現實情況：** 零售商不會在 UCP 和 ACP 之間選擇——他們會同時使用兩者。

| 零售商 | ChatGPT (ACP) | Gemini (UCP) |
|--------|---------------|--------------|
| Walmart | ✅ | ✅ |
| Shopify 商家 | ✅ (即將) | ✅ |
| Etsy | ✅ | ✅ |

### 6.2 整合策略矩陣

```
                    高整合複雜度
                         ▲
                         │
              ┌──────────┼──────────┐
              │          │          │
              │  同時實作 │  UCP     │
              │  UCP+ACP │  優先    │
              │          │          │
低交易量 ◄────┼──────────┼──────────┼────► 高交易量
              │          │          │
              │   觀望   │   ACP    │
              │          │  優先    │
              │          │          │
              └──────────┼──────────┘
                         │
                         ▼
                    低整合複雜度
```

### 6.3 建議行動

| 商家類型 | 建議 |
|----------|------|
| **已有 Stripe** | 立即啟用 ACP（一行代碼），觀望 UCP |
| **Shopify 商家** | 自動獲得 UCP 支持，考慮 ACP |
| **大型零售商** | 同時實作 UCP 和 ACP |
| **新進商家** | 優先 UCP（更廣泛的消費者觸及） |

## 九、前瞻觀點

### 9.1 協議融合趨勢

> 商家需要整合兩個協議，AI 代理需要在兩者間流暢運作，使**互通性**成為競爭性代理商務的新目標。

### 9.2 關鍵觀察

1. **不是零和遊戲**：UCP 和 ACP 服務不同的 AI 表面，支持兩者可最大化消費者觸及
2. **標準戰爭**：這是 Google 與 OpenAI/Stripe 在定義未來商務標準的競爭
3. **Amazon 的缺席**：值得注意的是，Amazon 尚未在任一陣營中扮演主導角色
4. **SEO 變革**：代理商務正在改變傳統 SEO 的遊戲規則

### 9.3 2026 年展望

- UCP 和 ACP 將持續演進並可能趨向更好的互通性
- 更多零售商將採用雙協議策略
- 支付處理商（如 Stripe 同時出現在兩陣營）將扮演橋樑角色
- MCP 和 A2A 作為基礎層將繼續被廣泛採用

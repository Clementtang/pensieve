# 標籤分類法 (Taxonomy)

> 標準化的標籤系統，確保內容分類一致性與可搜尋性

---

## 標籤使用原則

1. **一致性**：同一概念使用同一標籤（如「金融科技」而非「FinTech」或「fintech」）
2. **精準性**：選擇最能描述內容的標籤，避免過於籠統
3. **適量性**：每篇文章 3-7 個標籤為佳
4. **層級性**：優先使用主分類標籤，再加上特定標籤

---

## 標籤分類

### 一、內容類型標籤

標記文章的類型，**每篇文章必選一個**：

| 標籤 | 說明 | 適用場景 |
|------|------|---------|
| `企業研究` | 深度企業分析報告 | company-research 類文章 |
| `產業研究` | 產業/技術/市場分析 | industry-research, topic-research |
| `議題研究` | 複雜議題探討 | topic-research |
| `社群貼文` | 短篇社群媒體內容 | 適合分享的精簡文章 |
| `教學` | 步驟式操作指南 | tutorial 類文章 |
| `筆記` | 個人記錄 | note 類文章 |

### 二、產業/領域標籤

標記文章涉及的產業或領域：

| 標籤 | 涵蓋範圍 |
|------|---------|
| `AI` | 人工智慧、機器學習、LLM |
| `金融科技` | 支付、銀行、投資科技 |
| `電商` | 電子商務、零售平台 |
| `零售科技` | 智慧零售、POS、ESL |
| `企業軟體` | SaaS、ERP、CRM |
| `雲端服務` | 雲端基礎設施、IaaS、PaaS |
| `自動駕駛` | 自駕車、Robotaxi |
| `衛星網路` | 衛星通訊、太空科技 |
| `餐飲科技` | 餐飲業數位化 |
| `消費品牌` | D2C、消費品 |

### 三、主題標籤

標記文章探討的主題：

| 標籤 | 說明 |
|------|------|
| `併購` | M&A 分析 |
| `創業` | 創業故事、新創生態 |
| `投資` | 融資、估值、投資分析 |
| `競爭分析` | 競爭格局、市場競爭 |
| `地緣政治` | 國際關係對商業的影響 |
| `數位轉型` | 企業數位化轉型 |
| `平台競爭` | 平台經濟、生態系競爭 |
| `中國出海` | 中國企業國際化 |
| `台灣市場` | 台灣本地市場分析 |

### 四、公司/產品標籤

標記文章涉及的主要公司或產品（使用官方名稱）：

**科技巨頭：**
- `Google`, `Amazon`, `Meta`, `Apple`, `Microsoft`

**AI 公司：**
- `Anthropic`, `OpenAI`, `Claude`, `ChatGPT`

**金融科技：**
- `Airwallex`, `Stripe`, `Toast`

**其他知名公司：**
- `Shopify`, `Uber`, `Lyft`, `Tesla`

**使用原則：**
- 公司名稱使用英文原名
- 產品名稱使用官方命名
- 如有中文常用名，可並列（如 `瑞幸咖啡`）

## 標籤格式規範

### 語言選擇

| 類型 | 規範 | 範例 |
|------|------|------|
| 公司/產品名 | 英文原名 | `Airwallex`（非「空中雲匯」）|
| 產業/主題 | 中文 | `金融科技`（非「FinTech」）|
| 縮寫 | 保持常用形式 | `AI`, `SaaS`, `ESL` |

### 大小寫

- 英文標籤首字母大寫：`Anthropic`, `Claude`
- 縮寫全大寫：`AI`, `SaaS`, `POS`
- 中文標籤無此問題

### 標籤組合範例

**企業研究報告：**
```yaml
tags: ["企業研究", "Airwallex", "金融科技", "跨境支付"]
```

**社群貼文：**
```yaml
tags: ["社群貼文", "AI", "Claude", "Anthropic"]
```

**產業分析：**
```yaml
tags: ["產業研究", "零售科技", "ESL", "數位轉型"]
```

## 新增標籤流程

1. 檢查本文件是否已有適合的標籤
2. 如需新增，考慮：
   - 是否會重複使用（至少 3 篇以上）
   - 是否與現有標籤重疊
   - 應歸入哪個分類
3. 新增至本文件對應分類中
4. 更新 CHANGELOG.md

## 常見錯誤

❌ **不一致的命名：**
- `fintech` vs `FinTech` vs `金融科技` → 統一用 `金融科技`
- `AI編碼` vs `AI 編碼` → 統一用 `AI編碼`（無空格）

❌ **過於籠統：**
- 只用 `科技` → 應具體到 `AI` 或 `金融科技`

❌ **過於特定：**
- `2024年Q4財報分析` → 改用 `財務分析`

❌ **混用語言：**
- `人工智能` vs `AI` → 統一用 `AI`
